# Contributing to Legacy64

Thank you for helping preserve 64-bit computing history!

## How to Contribute

### Reporting Issues

If you spot an inaccuracy — wrong release year, incorrect socket name, missing hardware — please [open an issue](../../issues/new) with:

- The table or section where the error appears
- What is currently listed
- What the correct information should be
- A source or reference for the correction (datasheet, product page, archived press release, etc.)

### Adding New Entries

1. Fork this repository and create a feature branch.
2. Add your entry to the appropriate table in `README.md` (or create a new section if needed).
3. Keep the Markdown table formatting consistent with existing entries.
4. Open a Pull Request with a short description of what you added and why it belongs here.

### Style Guide

- Use plain Markdown tables for hardware and software lists.
- Dates should be the **release year** (four digits), not the announcement year where possible.
- CPU/chipset names should match the official product name used by the manufacturer.
- Links must point to stable, publicly accessible URLs (prefer `archive.org` snapshots for pages that may disappear).

## Code of Conduct

Be respectful. This is a technical preservation project — focus on facts and accuracy, not opinions about vendors or products.
